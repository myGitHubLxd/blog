# 最少位翻转算法

## 问题描述
一个特工向总部发送包含项目详情的秘密消息。他向机构的计算机(P)发送一份软拷贝，并通过传真向技术负责人Roger(Q)发送一份硬拷贝。但在传输过程中，网络中的噪声导致数据消息P的一些位发生扭曲。然而我们知道Roger总是匹配两个消息的二进制值，并检查是否可以通过翻转最少数量的位将消息P转换为消息Q。

**目标**：找到将消息P转换为消息Q所需翻转的最少位数。

## 算法思路

### 核心洞察
要将数字P转换为数字Q，我们需要翻转P和Q不同的那些位。这相当于计算P和Q之间的汉明距离(Hamming Distance)。

### 算法步骤：
1. **异或运算**：计算 `P ^ Q`（P和Q的异或）
   - 异或操作：相同位返回0，不同位返回1
   
2. **计算设置位**：统计异或结果中1的个数
   - 每个1位代表一个需要翻转的位置

### 示例：
```
P = 10 (二进制: 1010)
Q = 20 (二进制: 10100)

异或 = 1010 ^ 10100 = 11110
11110中1的个数 = 4

因此，最少需要翻转 4 位
```

## Swift实现

### 方法1：手动位计数
```swift
func minimumBitFlips(_ p: Int, _ q: Int) -> Int {
    // 异或运算得到P和Q之间不同的位
    let xorResult = p ^ q
    var count = 0
    var num = xorResult
    
    // 使用位操作计算设置位的个数
    while num != 0 {
        count += num & 1    // 检查最低位是否为1
        num >>= 1           // 右移一位，检查下一位
    }
    
    return count
}
```

### 方法2：使用内置函数
```swift
func minimumBitFlips(_ p: Int, _ q: Int) -> Int {
    let xorResult = p ^ q                    // 异或运算
    return xorResult.nonzeroBitCount        // 使用内置函数计算非零位数
}
```

## 复杂度分析
- **时间复杂度**：O(log(max(P, Q))) - 与位数成正比
- **空间复杂度**：O(1) - 常数额外空间

## 边界情况
- P = Q：返回0（无需翻转）
- P = 0 或 Q = 0：返回非零数中设置位的个数
- 负数：由于二进制补码表示，算法仍能正确工作

## 约束条件处理
算法高效处理给定约束条件：
- -10^4 ≤ num1, num2 ≤ 10^9
- 对正整数和负整数都能正确工作

## 算法原理详解

### 异或运算的特性
- 0 ^ 0 = 0（相同位为0）
- 0 ^ 1 = 1（不同位为1）
- 1 ^ 0 = 1（不同位为1）
- 1 ^ 1 = 0（相同位为0）

### 位操作技巧
1. `num & 1`：检查最低位是否为1
2. `num >>= 1`：右移一位，相当于除以2
3. `nonzeroBitCount`：Swift内置函数，直接计算非零位数

### 示例演示
```
P = 7  (二进制: 0111)
Q = 10 (二进制: 1010)

步骤1：异或运算
0111 ^ 1010 = 1101

步骤2：计算1的个数
1101 -> 1(第0位) + 0(第1位) + 1(第2位) + 1(第3位) = 3

结果：需要翻转3位
```
