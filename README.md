# 细胞自动机算法 (Swift实现)

## 问题描述

这是一个基于规则的细胞自动机算法，模拟由8个房屋组成的群体的状态变化。

### 规则说明

1. **初始状态**：8个单元格排成一行，每个单元格的状态为0（不活跃）或1（活跃）
2. **状态更新规则**：
   - 如果一个单元格的左右邻居状态相同（都是0或都是1），该单元格在下一天变为0（不活跃）
   - 如果一个单元格的左右邻居状态不同，该单元格在下一天变为1（活跃）
3. **边界处理**：最左边和最右边的单元格，其外侧邻居视为0

## 算法实现

### 核心函数

```swift
func cellCompete(_ states: [Int], _ days: Int) -> [Int]
```

**参数：**
- `states`: 初始状态数组，包含8个元素，每个元素为0或1
- `days`: 需要模拟的天数

**返回值：**
- 经过指定天数后的细胞状态数组

### 算法步骤

1. **输入验证**：如果天数为0，直接返回原状态
2. **状态复制**：创建当前状态的副本，避免修改原数组
3. **循环模拟**：对每一天进行状态更新
   - 创建新的状态数组存储下一天的状态
   - 遍历每个细胞位置
   - 获取左右邻居状态（边界处理）
   - 应用更新规则
   - 更新当前状态

## 使用示例

### 示例1：基本用法

```swift
let initialStates = [1, 0, 0, 0, 0, 1, 0, 0]
let days = 1
let result = cellCompete(initialStates, days)
// 结果：[0, 1, 0, 0, 1, 0, 1, 0]
```

**计算过程分析：**
```
初始状态: ● ○ ○ ○ ○ ● ○ ○
位置0: 左邻居=0, 右邻居=0, 相同 -> 下一天=0
位置1: 左邻居=1, 右邻居=0, 不同 -> 下一天=1
位置2: 左邻居=0, 右邻居=0, 相同 -> 下一天=0
位置3: 左邻居=0, 右邻居=0, 相同 -> 下一天=0
位置4: 左邻居=0, 右邻居=1, 不同 -> 下一天=1
位置5: 左邻居=1, 右邻居=0, 不同 -> 下一天=1
位置6: 左邻居=0, 右邻居=0, 相同 -> 下一天=0
位置7: 左邻居=0, 右邻居=0, 相同 -> 下一天=0
结果状态: ○ ● ○ ○ ● ● ○ ○
```

### 示例2：多天演化

```swift
let initialStates = [1, 1, 1, 0, 1, 1, 1, 1]
let days = 2
let result = cellCompete(initialStates, days)
```

## 算法复杂度

- **时间复杂度**：O(n × days)，其中n是细胞数量（本例中为8）
- **空间复杂度**：O(n)，需要额外的数组存储下一天的状态

## 扩展功能

算法可以轻松扩展以支持：
1. 不同数量的细胞
2. 不同的边界条件
3. 更复杂的状态更新规则
4. 可视化输出

## 注意事项

1. 输入数组必须只包含0和1
2. 边界细胞的外侧邻居始终视为0
3. 状态更新是同步进行的，即所有细胞同时更新到下一天的状态
